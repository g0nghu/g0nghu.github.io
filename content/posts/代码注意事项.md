# Python中类(class)的使用

- 1. 定义类的时候应该使用`self`关键字而不是类的名字, 典型的错误就是:
```python
class Block:
    
    def __init__(self, length, basis_size, operator_dict):
        Block.length = length
        Block.basis_size = basis_size
        Block.operator_dict = operator_dict
```
写出这种东西家里真要请哈基高了.

- 2. 关于`classmethod`和`staticmethod`
回答参考了[Meaning of @classmethod and @staticmethod for beginner](https://stackoverflow.com/questions/12179271/meaning-of-classmethod-and-staticmethod-for-beginner/12179752#12179752).
```python
class Date:
    def __init__(self, day=0, month=0, year=0)
        self.day = day
        self.month = month
        self.year = year

    @classmethod
    def from_string(cls, date_as_string):
        day, month, year = map(int, date_as_string.split('-'))
        date1 = cls(day, month, year)
        return date1
    
    @staticmethod
    def is_date_valid(date_as_string):
        day, month, year = map(int, date_as_string.split('-'))
        return day <= 31 and month <= 12 and year <= 3999
```

在`__init__()`中, 我们使用了`self`作为参数, 表示了对于一个新创建的**实例**的引用.

在`from_string()`中, 我们实际上想要把string改写成`Date`实例, 也就是说, 以一种不同的方式初始化一个`Date`实例. 但Python不支持重载, 取而代之的是提供了`cls`参数, 用来指代**类本身**, 而并非实例.

最后一个`is_date_valid()`, 不需要传递`cls`或者`self`作为参数, 但也不能访问某个`Date`实例中的参数和对应的方法. 这是一个按方法调用的函数, 调用请使用`Date.is_date_valid()`, 并不需要创建一个实例.


# PyTorch使用相关

- 1. 如果想把什么东西算进损失函数里, 要注意中间应该全部使用torch的操作, 这样可以保证每一步都可以使用torch的自动微分. 想算梯度就不要用numpy计算.
- 2. PyTorch感觉使用起来比TensorFlow拟人多了, 感觉更模块化了, 也没有占位符这种东西.

# Markdown相关

- 1. 我使用的hexo的markdown渲染器不支持`\ket{}`, 只能使用`\rangle`代替.
- 2. 写公式的时候, 有时太长要换行, 这时如果加号或者减号在开头, 后面不要打空格, 否则会优先判断成列表.
